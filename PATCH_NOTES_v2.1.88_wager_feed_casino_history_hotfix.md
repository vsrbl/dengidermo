# PATCH NOTES v2.1.88 — Wager / Feed / Casino History hotfix

Дата: 2026-07-06

## FIXES

- ROOM WAGER accept больше не имеет отдельного скрытого времени.
  - WAGER живёт только внутри обычного INSTALL-окна.
  - Когда INSTALL заканчивается, непринятый WAGER исчезает вместе с установкой.
- Кнопка `ACCEPT WAGER` больше не пересоздаётся каждый кадр.
  - Из-за старого rerender pointerdown происходил по одной кнопке, а click уже попадал в новый DOM-элемент.
  - Добавлен `pointerdown` accept и сохранение DOM до смены offer/active состояния.
  - Hover/click теперь должны реагировать нормально.
- История/feed теперь позиционируется динамически ниже всего левого верхнего HUD-блока.
  - Если есть призы контракта, feed опускается ниже них.
  - Выравнивание берётся от реального `hud-lefttop` rect, а не от фиксированной магической высоты.
- Текст приза контракта для reroll стал понятнее.
  - Было вроде: `ПЕРЕБРОС ВЫБОРА +2 АКТИВЕН x1`.
  - Теперь показывается человекочитаемо: `ПЕРЕБРОС ВЫБОРА + ещё 2 · 1 переброс`.
- Casino history/feed снова пишет подробный результат ставки.
  - Исправлена ошибка в `casinoResult()`: там использовалась локальная функция `name()`, которая существовала только внутри другого обработчика событий.
  - Из-за этого подробный feed казино мог падать и не появляться.
  - Теперь casino feed использует безопасное имя игрока из HUD name map.

## CHECKS

- `node --check` по `src`, `shared`, `server`, `config.js`.
- Импорт `shared/sim.v2-1.js`.
- Smoke-test ROOM WAGER: offer создаётся без `expires`, accept переводит offer в active.
