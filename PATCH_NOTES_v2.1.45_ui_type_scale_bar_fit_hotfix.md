# PATCH NOTES — v2.1.45 ui type scale bar fit hotfix

## Что исправлено
- Все явные CSS-размеры текста интерфейса увеличены на `1.25x`, чтобы меню, HUD, подсказки, таблицы, карточки и модальные окна читались крупнее.
- Нижний левый HUD с HP и EXP переразмечен под увеличенный текст.
- Полоска HP стала выше и шире, чтобы значения вроде `100 / 100` не упирались в рамку.
- Полоска EXP тоже стала выше: старые `10px` высоты были меньше высоты строки текста, из-за чего цифры визуально вылезали из полосы.
- Текст внутри HP/EXP теперь центрируется через flex-разметку, без зависимости от фиксированного `line-height`.
- Добавлены ограничения ширины для маленьких экранов, чтобы увеличенный HUD не ломал правую часть интерфейса.

## Версия
- `VERSION` → `v2.1.45`
- `BUILD_ID` → `ui_type_scale_bar_fit_hotfix`
- package version → `2.1.45`
- cache query strings в HTML обновлены до `2.1.45`

## Зачем
Предыдущая разметка была рассчитана на маленький текст. Особенно плохо это было видно на EXP: высота полосы была `10px`, а текст рисовался строкой около `15px`, поэтому цифры физически не могли поместиться внутрь. Теперь текст крупнее по всему интерфейсу, а проблемные полосы получили нормальную высоту и центрирование.
