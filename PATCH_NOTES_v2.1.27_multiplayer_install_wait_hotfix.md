# PATCH NOTES v2.1.27 — multiplayer install wait hotfix

## Исправлено
- Исправлено сломанное ощущение окна улучшений в мультиплеере: после выбора игрок больше не остаётся без понятного состояния.
- Если один игрок уже выбрал улучшение, а другие ещё выбирают, показывается отдельное окно ожидания.
- В окне ожидания видно, сколько игроков ещё выбирает и кого именно ждём.
- То же поведение работает для обычных улучшений и для сигнатур босса.
- Быстрый выбор клавишами и Esc больше не пытается выбирать вариант, когда открыто только окно ожидания.

## Сетевой слой
- В снимок комнаты добавлено безопасное состояние ожидания выбора: всего игроков, готовые игроки, ожидающие игроки, имена ожидающих и ближайший таймер автовыбора.
- Старые двойные подтверждения `offer_close` больше не должны визуально ломать окно: клиент сам показывает ожидание до перехода в следующую комнату.

## Проверка
- Проверен сценарий: 2 игрока, один выбрал улучшение, второй ещё выбирает — первый видит ожидание, комната не стартует раньше времени.
- Проверен сценарий: сигнатура босса в мультиплеере — ожидание работает так же.
