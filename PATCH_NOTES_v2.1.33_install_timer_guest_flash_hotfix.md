# PATCH NOTES v2.1.33 — install timer + guest flash hotfix

## Главное
- Починена фаза выбора улучшений в мультиплеере после v2.1.32.
- Таймер в окне выбора больше не сбрасывается от повторных сетевых сообщений.
- У гостя больше не должна вспыхивать лишняя плашка “выбери улучшение” до появления настоящих вариантов.

## Таймер выбора
- Повторная отправка того же самого предложения улучшения теперь не пересоздаёт окно.
- Если `offerId` и варианты совпадают, клиент только обновляет служебные данные, но не возвращает полоску таймера назад.
- Полоса таймера теперь убывает монотонно и не выглядит бесконечной.
- Сервер/хост теперь передаёт гостю оставшееся время и исходную длину предложения.

## Гостевой игрок
- Промежуточное окно синхронизации больше не появляется мгновенно перед настоящим выбором.
- Если варианты ещё не пришли, клиент сначала коротко ждёт настоящий `offer`.
- Экран синхронизации включается только если варианты реально задержались, а не на каждый нормальный сетевой кадр.
- Убран общий баннер перехода “ФАЗА УЛУЧШЕНИЙ / выбор улучшений”, который мог мелькать до реального окна выбора.

## Мультиплеер
- Состояние “выбрал / ждём остальных” сохранено.
- Повторная доставка предложений осталась, но больше не ломает видимый таймер.
- Обычные улучшения и сигнатуры босса используют одну и ту же защиту от сброса окна.

## Проверки
- `node --check server/index.js`
- `node --check shared/*.js src/*.js`
- import-smoke `protocol/sim/audio`
- smoke-тест фазы улучшений на 2 игрока
- `unzip -t`
