# v2.1.170 — active_preview_void_pathfinding

- Q/R протоколы с зоной применения теперь требуют подтверждения: первое нажатие показывает область, второе запускает эффект.
- VOID CUT переведён на построение от выбранной точки: первое Q ставит начальную точку, следующие нажатия подтверждают звенья по пунктирной линии.
- Пунктирная подсказка VOID CUT обновляется по направлению прицела и показывает номер текущего звена.
- Добавлена авторитетная подсказка радиуса/цели в снапшот симуляции, чтобы host и client видели одинаковую зону применения.
- Движение врагов получило лёгкую grid-навигацию поверх старого steerMove: мобы лучше обходят стены, сохраняя свои роли, дистанции и специальные поведения.
- R-протоколы с радиусом получили такой же шаг предпросмотра перед активацией.
- Добавлена отрисовка preview-зон и preview-линий в клиенте.

Проверка: node --check для JS, smoke import shared modules, базовый симуляционный smoke-test.
