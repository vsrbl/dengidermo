# PATCH NOTES v2.1.34 — guest data loading window

## Зачем
В мультиплеере у гостя после входа в фазу улучшений могло быть непонятное пустое/преждевременное состояние: игрок видел, что что-то происходит, но настоящие варианты ещё не пришли. Из-за этого можно было начать жать клавиши вслепую.

## Изменения
- Добавлено маленькое окно ожидания данных для гостевого игрока.
- До прихода настоящих вариантов оно показывает не `выбери улучшение`, а терминальный статус:
  - `ПОЛУЧЕНИЕ ДАННЫХ...`
  - `Синхронизация улучшений с ведущим узлом.`
  - `Не нажимай вслепую — варианты появятся сами.`
- Для сигнатур босса показывается отдельный вариант текста:
  - `ПОЛУЧЕНИЕ СИГНАТУРЫ...`
  - `Считываем пакет угрозы от узла ведущего.`
- У этого окна нет полосы таймера выбора, чтобы не было ощущения бесконечного или дёргающегося таймера.
- Когда настоящий список вариантов приходит, окно автоматически заменяется обычным выбором улучшений или сигнатур.
- Если игрок уже выбрал и ждёт остальных, старое окно ожидания остальных осталось отдельным и продолжает показывать список ожидающих игроков.

## Технически
- Добавлено отдельное состояние `dataLoading` для окна улучшений.
- Состояние получения данных не перезаписывается текстом `ВЫБЕРИ УЛУЧШЕНИЕ` на следующем кадре.
- Повторные сетевые снимки продолжают обновлять маленький экран ожидания, но не создают таймер выбора.

## Проверки
- `node --check` для `server/index.js`, `shared/*.js`, `src/*.js`.
- Import smoke: `protocol`, `data`, `sim`, `hud`.
- `unzip -t` для итогового архива.
