# v2.1.149 — process_controller_start_cmd_process_life_sfx_hotfix

Контролёр процессов приведён ближе к целевой роли: стартует только с CMD, не получает CTRL/QRN-прокачки в установках, а подконтрольные процессы больше не меняют размер.

## Gameplay
- Контролёр стартует только с одной командой: `CMD / КОМАНДА ЗАХВАТА`.
- `QRN / КАРАНТИННЫЙ ЯКОРЬ` и `SAW / РАЗБОР ПРОЦЕССА` теперь открываются через ящик команд/WPN, а не лежат в стартовом наборе.
- CTRL/QRN-улучшения убраны из окна установок: они относятся к WPN/ящику команд.
- Добавлены WPN-улучшения Контролёра:
  - `CTRL: СРОК +` — подконтрольные процессы живут дольше.
  - `CTRL: ПЕРЕНОС` — процессы не очищаются у портала и аккуратно переносятся в следующий сектор.

## Controlled processes
- При захвате процесс получает полный срок контроля.
- При новом захвате срок контроля всех уже захваченных процессов сбрасывается на максимум.
- Срок контроля теперь реально утекает и отображается в CTRL-HUD.
- При обычном переходе через портал подконтрольные процессы очищаются.
- С `CTRL: ПЕРЕНОС` процессы сохраняются, переносятся рядом с игроком и получают безопасные позиции/сброшенное состояние.
- Убран визуальный shrink для подконтрольного `RUN`: размер теперь совпадает с исходной угрозой.

## Audio/feedback
- Добавлены отдельные SFX-события для:

  - успешного захвата;
  - приказа ПКМ;
  - SAW;
  - постановки QRN;
