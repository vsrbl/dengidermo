# PATCH NOTES — v2.1.32 — multiplayer install phase repair

## Зачем
Мультиплеерная фаза улучшений могла выглядеть сломанной: один игрок уже выбрал, другой ещё выбирает, а окно у первого оставалось в странном состоянии или ждало неочевидно. Особенно это проявлялось после правок ожидания и отдельного меню сигнатур босса.

## Исправлено
- Фаза улучшений теперь восстанавливается по общему снимку комнаты, а не только по разовым `offer/offer_close` сообщениям.
- Если игрок уже выбрал, а другие ещё выбирают, окно явно показывает ожидание остальных.
- Если клиенту нужно выбрать улучшение, но сетевое предложение ещё не дошло, показывается отдельное состояние синхронизации вместо пустого/сломавшегося окна.
- Если игрок отвалился во время фазы улучшений, его старое предложение больше не держит всю комнату в ожидании.
- Хост теперь периодически переотправляет активное предложение улучшения, чтобы клиент мог восстановиться после неправильного порядка сетевых сообщений.
- Если у игрока несколько накопленных улучшений, следующее предложение отправляется сразу после выбора предыдущего.

## Затронуто
- Обычное окно улучшений.
- Окно сигнатур босса.
- Сетевое ожидание игроков в фазе `install`.

## Не менялось
- Баланс улучшений.
- Состав сигнатур босса.
- Боевая симуляция.
- Протокол снимков состояния не повышался.

## Проверки
- `node --check` для `local`, `hud`, `sim`, `protocol`, `main`.
- Smoke-тест обычных улучшений на 2 игрока.
- Smoke-тест сигнатур босса на 2 игрока.
- Smoke-тест локальной комнаты с хостом и гостем.
- `unzip -t` финального архива.
