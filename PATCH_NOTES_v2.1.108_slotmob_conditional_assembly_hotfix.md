# v2.1.108 — Slot Mob Conditional Assembly Hotfix

## Главное
- Casino slot mob больше не создаётся заранее как скрытый enemy.
- Во время поломки/разлёта/магнитной сборки в `run.enemies` нет `slot_mob` вообще.
- Сборка из 4 блоков теперь является реальным условием появления моба.

## Цепочка появления
- Casino overload создаёт только `pendingSlotMobs` и визуальный `casino_overload` fx.
- 4 цветных блока разлетаются, лежат, магнитятся к центру и дают поэтапные impacts.
- После финального impact есть небольшой post-buffer, чтобы блоки успели исчезнуть.
- Только после этого вызывается `createSlotMobEntity()` и появляется настоящий enemy `slot_mob`.
- Roll внутри моба начинается только после создания сущности.

## Возрождение между жизнями
- При промежуточной смерти slot mob удаляется из `run.enemies`.
- Затем создаётся pending-сборка из 4 блоков.
- Новый slot mob появляется только после финальной стыковки 4-го блока.

## Техническое
- Добавлен `pendingSlotMobs` в run state.
- `roomHasLiveEnemies()` и `liveEnemyCount()` учитывают pending-сборку, чтобы портал не открылся, пока mob ещё должен собраться.
- Убран ранний hidden/non-physical enemy как источник рассинхрона визуала и логики.
